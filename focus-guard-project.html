Focus Guard: A Full-Stack IoT Productivity System

A tangible IoT device and native Android app I built from scratch to automatically manage a phone's 'Do Not Disturb' (DND) mode and track focus statistics.

The Problem, The Solution

In today's world, digital distractions are the #1 killer of productivity. The constant stream of notifications breaks our concentration and makes deep work nearly impossible. Software-only timers are easily ignored and still require interacting with the source of distractionâ€”the phone.

To solve this, I designed and built Focus Guard, a complete, end-to-end system. It combines a physical, tangible timer with a native Android app. A single button press on the device creates a "distraction-free bubble" by automatically sealing the phone in 'Do Not Disturb' mode.

Key Features

Automatic DND Control

A ForegroundService automatically activates/deactivates DND mode for a truly distraction-free environment.

Real-Time Hardware Sync

An ESP8266 web server allows the app and device to stay perfectly in sync. Control from one is reflected on the other.

Productivity Statistics

Completed sessions are saved to a local Room Database and visualized in a bar chart using MPAndroidChart.

Tangible Interface

A custom device with an OLED menu, RGB status light, and audio alerts provides a physical way to manage time.

Technical Architecture

Part 1: The Hardware & Enclosure

I chose the ESP8266 for its built-in Wi-Fi and low cost. An OLED display provides visual feedback, while a buzzer and RGB LED give audio and status alerts. I designed and built a custom enclosure to house all components, creating a polished, final product complete with a QR code for app download.

Part 2: The Firmware (ESP8266)

The device's brain was written in C++ (Arduino) and implements a robust state machine (IDLE, FOCUS, BREAK). It runs an ESP8266WebServer that serves a RESTful API to the app. I optimized the /status endpoint for memory safety by using ArduinoJson with a static char buffer, preventing heap fragmentation and ensuring long-term stability.

Part 3: The Native Android App (Kotlin)

The app is built natively in Kotlin using Material Design 3. It communicates with the device's API using the Volley library. To ensure DND control and polling continued even when the app was in the background, I architected it around a persistent ForegroundService. All statistics are saved locally using a Room Database.
